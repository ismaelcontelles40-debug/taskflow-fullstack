# Metodologías de desarrollo

---

# ¿Qué es Agile?

Agile es una metodología de trabajo utilizada en el desarrollo de software que busca crear aplicaciones de forma más flexible, rápida y organizada.

Su principal objetivo es mejorar la capacidad de adaptación ante cambios durante el desarrollo del proyecto.

En lugar de esperar meses para entregar un producto completo, Agile divide el trabajo en pequeñas partes funcionales que se desarrollan de manera progresiva.

Gracias a esto:

- Se detectan errores antes
- Se mejora la comunicación del equipo
- El cliente puede dar feedback continuamente
- El desarrollo es más organizado y flexible

---

# ¿Qué es Scrum?

Scrum es un framework basado en Agile que organiza el trabajo en ciclos cortos llamados sprints.

Cada sprint suele durar entre una y cuatro semanas y durante ese tiempo el equipo desarrolla una parte concreta del proyecto.

Scrum utiliza varios conceptos importantes:

## Roles principales

### Product Owner

Se encarga de decidir qué funcionalidades son más importantes para el proyecto.

### Scrum Master

Ayuda al equipo a seguir correctamente la metodología Scrum y elimina posibles bloqueos.

### Equipo de desarrollo

Son las personas encargadas de programar y desarrollar la aplicación.

---

# Conceptos principales de Scrum

## Sprint

Periodo corto de trabajo donde se desarrolla una parte del proyecto.

## Product Backlog

Lista general de tareas y funcionalidades pendientes.

## Sprint Backlog

Conjunto de tareas seleccionadas para un sprint concreto.

## Sprint Review

Reunión final donde se muestra el trabajo realizado.

## Sprint Retrospective

Reunión donde el equipo analiza qué ha funcionado bien y qué puede mejorar.

---

# ¿Qué es Kanban?

Kanban es un método visual para organizar tareas y controlar el flujo de trabajo.

Se basa en un tablero dividido en columnas donde las tareas se van moviendo según su estado.

Las columnas más habituales son:

- Backlog
- Todo
- In Progress
- Review
- Done

Kanban permite visualizar fácilmente el estado del proyecto y detectar bloqueos rápidamente.

---

# Diferencias entre Scrum y Kanban

| Scrum | Kanban |
|---|---|
| Trabaja mediante sprints | Flujo continuo de trabajo |
| Tiene roles definidos | No necesita roles específicos |
| Las tareas se planifican por sprint | Las tareas se gestionan continuamente |
| Más estructurado | Más flexible |
| Ideal para proyectos grandes | Ideal para mantenimiento o tareas continuas |

---

# ¿Cuándo utilizar cada metodología?

## Cuándo usar Scrum

Scrum es recomendable cuando:

- El proyecto es grande
- Hay varias personas trabajando
- Se necesita planificación organizada
- Existen entregas periódicas

---

## Cuándo usar Kanban

Kanban es recomendable cuando:

- El flujo de trabajo cambia constantemente
- Se necesita flexibilidad
- El equipo quiere visualizar tareas rápidamente
- Hay mantenimiento continuo de aplicaciones

---

# Conclusión

Las metodologías Agile ayudan a mejorar la organización y el desarrollo de software.

Scrum permite trabajar de forma estructurada mediante sprints, mientras que Kanban ofrece una organización más visual y flexible.

Ambas metodologías son muy utilizadas actualmente en empresas tecnológicas porque mejoran la productividad, la colaboración y la capacidad de adaptación durante el desarrollo de proyectos.
